## 成本優化策略

### 策略 1：環境分離

**問題：** 開發環境也用生產級架構？

**最佳實踐：**

```
開發環境：
- 單 AZ（減半資源）
- 單 NAT Gateway
- 最小實例（t3.nano）
- 需要時開啟，不用時關閉

測試環境：
- 單 AZ
- NAT Gateway
- 小實例
- 定期運行

生產環境：
- 2+ AZ
- 2 NAT Gateway
- 適當大小的實例
- 24/7 運作
```

**成本對比：**
```
環境不一致：
開發 = 生產 = $100/月
總計：$200/月

環境分離：
開發 = $25/月
生產 = $100/月
總計：$125/月

節省：$75/月 = $900/年
```

---

### 策略 2：智能啟動

**使用 Auto Scaling：**

```
工作時間 (8:00-18:00):
→ 2 個實例處理流量

下班時間 (18:00-8:00):
→ 縮減到 1 個實例

深夜 (23:00-6:00):
→ 縮減到 0 個實例（使用 Lambda）

成本節省：
- 16 小時只 1 個實例：節省 50%
- 7 小時 0 個實例：節省 100%
→ 總節省約 65%
```

**實作範例：**
```hcl
resource "aws_autoscaling_group" "app" {
  min_size = 0
  max_size = 10
  desired_capacity = 2

  # 按時間表調整
  # 工作時間：2 實例
  # 其他時間：0-1 實體
}
```

---

### 策略 3：使用 Spot 實例

**什麼是 Spot 實例？**
- AWS 閒置的閒置容量
- 可節省 50-90%
- 可能被中斷

**適合場景：**

| 使用情境 | 適合 Spot | 原因 |
|---------|----------|------|
| **批次處理** | ✅ | 可以重試 |
| **模型訓練** | ✅ | 可以暫停 |
| **資料分析** | ✅ | 可中斷處理 |
| **API 服務** | ❌ | 不能中斷 |
| **資料庫** | ❌ | 必須持續運行 |

**成本節省範例：**
```
On-Demand t3.medium: $0.0416/小時
Spot t3.medium: $0.0125/小時 (省 70%)

24/7 運行：
On-Demand: $30/月
Spot: $9/月

節省：$21/月 = $252/年
```

---

### 策略 4：預留實例（Reserved Instances）

**什麼是預留實例？**
- 承諾使用 1-3 年
- 獲得大幅折扣
- 適合穩定工作負載

**成本比較：**

| 實例類型 | On-Demand | 1 年預留 | 3 年預留 |
|---------|-----------|----------|----------|
| t3.micro | $0.0084 | ~57% 折扣 | ~70% 抃扣 |
| t3.small | $0.020 | ~58% 折扣 | ~70% 折扣 |
| t3.medium | $0.0416 | ~54% 折扣 | ~66% 折扣 |

**實際計算：**
```
t3.medium 24/7 運行：
On-Demand: $30/月
1 年預留: $13/月 (省 $17/月)
3 年預留: $10/月 (省 $20/月)

年節省：
1 年：$17 × 12 = $204
3 年：$20 × 36 = $720
```

## 各組件的成本詳細分析

### NAT Gateway 成本

```
小時費率：$0.045
每天：$0.045 × 24 = $1.08
每月：$1.08 × 30 = $32.40
每年：$32.40 × 12 = $388.80

多 AZ 的影響：
2 AZ：$32.40 × 2 = $64.80/月
3 AZ：$32.40 × 3 = $97.20/月
```

**成本優化技巧：**
```
1. 只在需要的 AZ 部署
   → 如果只用 2 個 AZ，就不需要 3 個 NAT Gateway

2. 考慮單 NAT + 共享子網
   → 所有私有子網共用一個 NAT
   → 省一半成本，犧牲一些可用性

3. 使用 NAT Instance（開發環境）
   → 開發時省錢
   → 生產時用 NAT Gateway
```

---

### EC2 成本

**不同實例類型的月成本：**

| 實例 | vCPU | 記憶 | 每小時 | 24/7 月成本 |
|------|-----|------|--------|-----------|
| t3.nano | 1 | 0.5G | $0.0058 | ~$4 |
| t3.micro | 1 | 1G | $0.0084 | ~$6 |
| t3.small | 1 | 2G | $0.020 | ~$15 |
| t3.medium | 2 | 4G | $0.0416 | ~$30 |
| t3.large | 2 | 8G | $0.0832 | ~$60 |
| m5.large | 2 | 8G | $0.096 | ~$70 |

**選擇依據：**
- 開發/測試：t3.nano 或 t3.micro
- API 服務：t3.small 或 t3.medium
- AI 推理：t3.medium 或 t3.large（需 GPU）
- 生產環境：t3.large 或更大

**AI 應用的實例選擇：**
```
輕量級 API:
→ t3.small ($15/月)
→ CPU 足夠，成本合理

中量級推薦:
→ t3.medium ($30/月)
→ 需要快取可以考慮

大型模型訓練:
→ g4dn.xlarge (GPU)
→ 但不在 Stage 1 範圍
```

### 資料傳輸成本

**Amazon EC2 的資料傳輸：**

| 方向 | 成本 | 說明 |
|------|------|------|
| **傳入** | $0.01/GB | 資料傳入 EC2 |
| **傳出** | $0.01/GB | 資料傳出 EC2 |
| **NAT 傳出** | $0.045/GB | 經過 NAT 傳出 |

**實例計算：**

```
場景：AI 應用每天處理 100 個請求
- 每個請求上傳 2KB（文字）
- 每個回應下載 50KB（AI 回應）
- 每天：100 × (2+50)KB = 5.2MB
- 每月：5.2MB × 30 = 156MB

資料傳入：0.01 × 156MB / 1024 = $0.0015
資料傳出：0.01 × 156MB / 1024 = $0.0015
總計：~$0.003/月（幾乎可以忽略）
```

**高流量場景：**
```
場景：AI 應用每天處理 1TB 資料
資料傳出：0.01 × 1024GB / 1024 = $10.24/天
資料傳入：0.01 × 1024GB / 1024 = $10.24/天
總計：$20.48/天 = $614/月

注意：這只是資料傳輸，不包含計算成本
```

## 成本優化的實際應用

### 案例 1：學習專案

**需求：**
- 學習 Terraform 和 AWS
- 不需要 24/7 運行
- 預算有限（每月 $20 以內）

**最佳設計：**
```
VPC: 10.0.0.0/16
子網：1 公有 + 1 私有（單 AZ）
EC2: t3.nano（按需）
NAT: 不使用 NAT（實驗時不用）
實際執行：只在學習時開啟
→ 開啟 50 小時/月，成本約 $0.25
```

**成本：**
- VPC：$0
- 子網：$0
- NAT：$0
- EC2：$0.25/月
- **總計：<$1/月**

---

### 案例 2：個人 AI 副業專案

**需求：**
- 個人 AI 應用
- 對日流量：1,000 次請求
- 7天服務，營業時間
- 預算：每月 $50

**最佳設計：**
```
VPC: 10.0.0.0/16
子網：1 公有 + 1 私有（單 AZ）
EC2: t3.micro + Auto Scaling
- 工作時間（8 小時）：1 實例
- 其他時間：0 實例
NAT: 使用 NAT Instance（開發環境成本）

Auto Scaling 節省：
- 每天運行 8 小時：$2/月
- 不運行時：$0
→ 較 24/7 運行省 $4/月
```

**成本：**
- VPC：$0
- 子網：$0
- NAT Instance：$15/月
- EC2：$2/月（使用 Auto Scaling）
- 其他：$5/月
- **總計：~$22/月**

---

### 案例 3：初創公司生產環境

**需求：**
- 需要高可用性
- 24/7 運作
- 日流量：10,000 請求
- 成長潛力大

**最佳設計：**
```
VPC: 10.0.0.0/16
子網：2 公有 + 2 私有 + 2 資料（3 AZ）
EC2: t3.small + Auto Scaling
- 基礎：1 實體/AZ
- 尖峰：3 實體/AZ
- 過夜：1 實體/AZ
NAT: 2 NAT Gateway（高可用）

成本優化：
- 使用 Reserved Instances（1 年）
- 使用 Auto Scaling 縮減成本
- 預留實例處理尖峰流量
```

**成本：**
- VPC：$0
- NAT Gateway：$64/月
- EC2：
  - 基礎：$15 × 6 = $90/月
  - 尖峰：按需實例
  - Reserved：省 50%
  - Auto Scaling：省 40%（非工作時間）
  → 實際約 $60-80/月
- 其他：$20/月
- **總計：~$150/月**

## 隱藏成本警告

### 容易忽略的成本

**1. 資料庫成本（未在 Stage 1）：**

```
RDS t3.micro:
- 月固定成本：$15
- 儲存：$0.115/GB-month
- 100GB：$11.5/月
- 總計：$26.5/月

RDS t3.medium:
- 月固定成本：$135
- 儲存：100GB：$11.5/月
- 多 AZ 傳用：翻倍
- 總計：~$290/月
```

**2. OpenSearch/Elasticsearch（未在 Stage 1）：**

```
t3.small.search:
- 月固定成本：$70
- 單 AZ：$70
- 多 AZ：×2
- 總計：$140-280/月
```

**3. 負載平衡器：**

```
Application Load Balancer：
- 按小時費用：$0.0225
- LCU 費用：$0.008/LCU
- 1M LCU/月：$8

24/7 運行：
- 固定成本：$0.0225 × 24 × 30 = $16.2/月
- LCU 變動：依流量
- 總計：$20-50/月
```
