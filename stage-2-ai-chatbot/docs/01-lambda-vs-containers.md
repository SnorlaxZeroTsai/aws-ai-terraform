## 方案 1：AWS Lambda (Serverless)

### 架構

```
[API Gateway]
    ↓
[Lambda Function]
    ├─ Runtime (Python/Node.js/Go/Java)
    ├─ Handler Code
    ├─ Dependencies
    └─ Environment Variables
    ↓
[AWS Services Integration]
    ├─ DynamoDB
    ├─ S3
    ├─ Bedrock
    └─ 等...
```

### 計費詳解

**Lambda 定價結構：**

```
1. 請求費：
   - 免費額度：1M 請求/月
   - 超過：$0.20 per 1M 請求

2. 執行時間費：
   - 免費額度：400,000 GB-秒/月
   - 超過：$0.0000166667 per GB-秒
   - 公式：記憶體(GB) × 執行時間(秒) × 請求數

3. 資料傳輸：
   - 免費額度：1GB/月
   - 超過：$0.09 per GB
```

**實際計算範例：**

```
場景：AI Chatbot API
- 每月 100 萬請求
- 平均執行時間：3 秒
- 記憶體配置：512MB

計算：
請求費：
100 萬請求在免費額度內 = $0

執行費：
512MB = 0.5GB
0.5GB × 3s × 1,000,000 = 1,500,000 GB-秒

免費額度：400,000 GB-秒
超過：1,500,000 - 400,000 = 1,100,000 GB-秒

費用：1,100,000 × $0.0000166667 = $18.33

總成本：$0 + $18.33 = $18.33/月
```

### 優點

**1. 零運維**
```
你不需要管理：
❌ OS 更新
❌ 安全修補
❌ 容量規劃
❌ 負載均衡
❌ 健康檢查
❌ 日誌輪轉
❌ 監控設置

AWS 全部自動處理
```

**2. 自動擴展**
```
流量從 10 → 10,000 請求/秒
  ↓
Lambda 自動擴展
  ↓
無需任何配置
```

**3. 按使用付費**
```
沒有請求時：
→ 成本 = $0

有請求時：
→ 成本 = 請求數 × 單價
```

**4. 快速迭代**
```
程式碼修改：
  ↓
部署（秒級）
  ↓
立即生效
```

### 缺點

**1. 執行時間限制**
```
最大執行時間：15 分鐘

限制：
❌ 長時間批次處理
❌ 大檔案處理
❌ 影片編碼
❌ 機器學習訓練
```

**2. 冷啟動延遲**
```
第一次請求：
→ 100-500ms 延遲
→ AWS 準備執行環境
→ 載入程式碼

影響：
→ 間歇性流量延遲高
→ 用戶體驗受影響
```

**3. 狀態管理複雜**
```
無法使用：
❌ 本地檔案系統（/tmp 限制 512MB）
❌ 本地記憶體（每次執行重置）
❌ 持久連線

需要：
→ 外部狀態儲存（DynamoDB, S3, EFS）
```

**4. 除錯困難**
```
本地除錯：
→ 需要特殊工具（SAM CLI）
→ 環境不完全相同

生產除錯：
→ 只能依賴 CloudWatch Logs
→ 無法直接連線
```

### 適用場景

**✅ 完美適合：**
- HTTP API（秒級回應）
- 事件驅動處理
- 資料轉換
- 輕量級批次處理
- Webhook 處理
- 定期任務

**❌ 不適合：**
- 長時間任務（>15 分鐘）
- 需要持久連線
- 需要特殊硬體（GPU）
- 需要完整 OS 控制
- 高頻率、低延遲需求

## 方案 2：ECS Fargate (容器)

### 架構

```
[ECS Cluster]
    ↓
[ECS Task Definition]
    ├─ Container Image (Docker)
    ├─ CPU: 0.5 vCPU
    ├─ Memory: 1GB
    └─ Environment Variables
    ↓
[Fargate Instance]
    ├─ AWS 管理
    ├─ 無需管理 EC2
    └─ 按任務付費
```

### 計費詳解

**Fargate 定價結構：**

```
1. CPU 費用：
   - us-east-1: $0.04034 per vCPU-hour
   - 公式：vCPU × 運行小時 × $0.04034

2. 記憶體費用：
   - us-east-1: $0.0044 per GB-hour
   - 公式：GB × 運行小時 × $0.0044

3. 最小計算單位：1 分鐘
   - 不足 1 分鐘按 1 分鐘計算
```

**實際計算範例：**

```
場景：AI Chatbot API
- 2 個任務（2 個容器）
- 每個任務：0.5 vCPU + 1GB 記憶體
- 24/7 運行
- 100 萬請求/月

計算：
CPU：
0.5 vCPU × 24 小時 × 30 天 × 2 任務 = 720 vCPU-小時
費用：720 × $0.04034 = $29.04

記憶體：
1GB × 24 小時 × 30 天 × 2 任務 = 1,440 GB-小時
費用：1,440 × $0.0044 = $6.34

總成本：$29.04 + $6.34 = $35.38/月
```

### 優點

**1. 無需管理 EC2**
```
傳統 ECS：
→ 管理 EC2 叢集
→ 處理 Auto Scaling
→ 維護 Agent

Fargate：
→ AWS 管理
→ 只需定義任務
```

**2. 更好的資源控制**
```
精確指定：
→ CPU：0.25, 0.5, 1, 2, 4 vCPU
→ 記憶體：0.5GB, 1GB, 2GB, 4GB...
→ 確保資源隔離
```

**3. 支援長時間任務**
```
無執行時間限制
✅ 影片處理
✅ 大檔案轉換
✅ 批次分析
```

**4. 狀態管理**
```
容器內可以：
✅ 使用本地檔案系統
✅ 保持記憶體狀態
✅ 持久連線
✅ 複雜的依賴
```

### 缺點

**1. 按小時計費**
```
即使沒有請求：
→ 只要任務運行，就付費
→ 閒置時間也要付錢

範例：
→ 24/7 運行 = $35/月（不管有沒有流量）
```

**2. 需要容器知識**
```
需要學習：
→ Docker 基礎
→ Dockerfile 撰寫
→ 容器 Registry（ECR）
→ 映像管理
```

**3. 擴展不如 Lambda 快**
```
Lambda：
→ 毫秒級擴展

Fargate：
→ 秒級到分鐘級擴展
→ 需要啟動容器
```

**4. 運維複雜度**
```
需要管理：
→ 任務定義
→ 服務配置
→ Auto Scaling 規則
→ 負載均衡
→ 健康檢查
```

### 適用場景

**✅ 完美適合：**
- 長時間運行的服務
- 需要狀態的應用
- 複雜的依賴需求
- 批次處理任務
- 微服務架構
- 需要持久連線

**❌ 不適合：**
- 間歇性、短時間任務
- 快速原型開發
- 無容器知識的團隊

## 方案 3：EC2 (虛擬機)

### 架構

```
[EC2 Instance]
    ├─ OS (Amazon Linux 2023)
    ├─ Runtime (Python/Node.js/Go)
    ├─ Application
    └─ Dependencies
    ↓
[Auto Scaling Group]
    ↓
[Load Balancer]
```

### 計費詳解

**EC2 定價結構：**

```
1. 實例費用：
   - t3.medium: $0.0416 per hour
   - 公式：每小時費用 × 運行小時

2. 三種購買選項：
   a) On-Demand（按需）
      - $0.0416/hour
      - 靈活性高

   b) Reserved（預留）
      - 1 年：省 57%
      - 3 年：省 70%

   c) Spot（競價）
      - 省最多 90%
      - 可能被中斷
```

**實際計算範例：**

```
場景：AI Chatbot API
- 2 個 EC2 (t3.medium)
- 24/7 運行
- 100 萬請求/月

On-Demand 計算：
2 × $0.0416 × 24 × 30 = $59.90/月

Reserved (1 年) 計算：
$59.90 × (1 - 0.57) = $25.73/月

Spot 計算：
$59.90 × (1 - 0.70) = $17.97/月
```

### 優點

**1. 完全控制**
```
你可以控制：
✅ OS 級別設定
✅ 內核參數
✅ 特殊驅動
✅ 網路配置
✅ 安全設定
✅ 任何軟體
```

**2. 無執行限制**
```
無限制：
✅ 執行時間
✅ 記憶體使用
✅ 儲存空間
✅ 網路連線
```

**3. 可預測效能**
```
固定資源：
→ CPU 不會被其他用戶搶佔
→ 記憶體固定分配
→ 效能穩定
```

**4. 成本最佳化選項**
```
高流量場景：
→ Reserved Instances（省 70%）
→ Spot Instances（省 90%）
→ 實際成本可能低於 Lambda
```

### 缺點

**1. 高運維負擔**
```
你需要管理：
❌ OS 更新
❌ 安全修補
❌ 容量規劃
❌ Auto Scaling
❌ Load Balancer
❌ 健康檢查
❌ 日誌管理
❌ 監控設置
```

**2. 為閒置付費**
```
24/7 運行：
→ 不管有沒有請求
→ 都要付費

範例：
→ 流量 = 0
→ 成本 = $60/月（不變）
```

**3. 擴展慢**
```
Auto Scaling：
→ 偵測流量：1-2 分鐘
→ 啟動實例：2-5 分鐘
→ 應用準備：1-2 分鐘
→ 總計：5-10 分鐘

可能已經超時了
```

**4. 固定成本**
```
即使不用：
→ 預留實例要付費
→ 不能隨意停止
→ 預先承諾 1-3 年
```

### 適用場景

**✅ 完美適合：**
- 持續高流量（>70% 使用率）
- 需要完整 OS 控制
- 特殊硬體需求（GPU）
- 長時間運行的任務
- 複雜的應用架構
- 合規要求（需要控制）

**❌ 不適合：**
- 間歇性流量
- 快速迭代開發
- 無運維團隊
- 預算有限的 MVP

## AI 應用的選擇指南

### 場景 1：輕量級 AI API

**需求：**
- 文字生成（Claude 3 Haiku）
- 每個請求 < 5 秒
- 間歇性流量

**推薦：Lambda**

```
成本：~$20/月
理由：
✅ 請求時間短
✅ 流量不穩定
✅ 無需狀態
✅ 零運維
```

### 場景 2：中量級 AI 推理

**需求：**
- Embedding 生成
- 批次處理
- 需要快取

**推薦：ECS Fargate**

```
成本：~$50/月
理由：
✅ 可以使用本地快取
✅ 支援批次處理
✅ 執行時間不受限
✅ 可以使用 GPU
```

### 場景 3：重量級 AI 訓練

**需求：**
- 模型訓練
- 需要GPU
- 長時間運行

**推薦：EC2 (GPU)**

```
成本：~$500/月
理由：
✅ 需要 GPU
✅ 長時間運行
✅ 完全控制
✅ 可用 Spot 實例省錢
```

### 場景 4：混合架構

**需求：**
- 輕量級 API + 重量級處理

**推薦：Lambda + ECS**

```
[API Gateway]
    ↓
[Lambda] (驗證、路由)
    ↓
[ECS Fargate] (AI 推理)
    ↓
[模型服務]

成本：~$70/月
理由：
✅ 分層處理
✅ 最佳化成本
✅ 各司其職
```

## 成本轉換點計算

### 計算公式

```
Lambda 成本 = 請求費 + 執行費
執行費 = GB × 秒 × 請求數 × $0.0000166667

Fargate 成本 = (CPU費 + 記憶體費) × 小時
CPU 費 = vCPU × $0.04034
記憶體費 = GB × $0.0044

EC2 成本 = 實例費 × 小時
```

### 轉換點範例

```
場景：AI Chatbot
- 平均執行：3 秒
- 記憶體：512MB

Lambda vs Fargate：

每月請求數 | Lambda | Fargate | 較便宜
-----------|--------|---------|--------
10K        | $0.50  | $35     | Lambda
100K       | $1.83  | $35     | Lambda
1M         | $18.33 | $35     | Lambda
2M         | $36.66 | $35     | Fargate
5M         | $91.65 | $35     | Fargate

轉換點：~1.5M 請求/月

低於轉換點：Lambda 較便宜
高於轉換點：Fargate 較便宜
```

## 決策流程圖

```
開始
  ↓
[執行時間 > 15 分鐘？]
├─ Yes → ECS Fargate 或 EC2
└─ No → 繼續
    [需要 GPU 或特殊硬體？]
    ├─ Yes → EC2
    └─ No → 繼續
        [需要持久狀態/連線？]
        ├─ Yes → ECS Fargate
        └─ No → 繼續
            [月請求 < 1.5M？]
            ├─ Yes → Lambda
            └─ No → 考慮成本對比
```
